# Run With Love — сайт (всё в одном)

Один Django-проект, который отдаёт **и сайт, и админку, и API** на одном адресе.
Стек: Python 3 · Django 5 · Django REST Framework · SQLite · Django Admin.
React-фронт собран в статику и лежит в `static_site/` — отдаётся самим Django.

Сайт не собирает персональные данные: нет регистрации, форм и платежей —
все действия пользователей ведут на внешние сервисы по ссылкам (russiarunning,
добро.рф, ВКонтакте).

## Быстрый старт (одна команда для запуска)

```bat
cd backend
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
copy .env.example .env
python manage.py migrate
python manage.py createsuperuser
python manage.py seed_demo      :: необязательно — демо-данные
python manage.py runserver
```

Готово. Всё на одном адресе:

- Сайт: http://127.0.0.1:8000/
- Админка: http://127.0.0.1:8000/admin/
- API: http://127.0.0.1:8000/api/

> На Linux/Mac активация venv: `source .venv/bin/activate`.

## Публичное API (read-only, без авторизации)

| Endpoint | Описание |
|---|---|
| `GET /api/races/` | Забеги (с дистанциями, партнёрами, благополучателями) |
| `GET /api/news/` | Новости (с галереей изображений) |
| `GET /api/partners/` | Партнёры / спонсоры |
| `GET /api/beneficiaries/` | Благополучатели (приюты, фонды) |
| `GET /api/galleries/` | Фотоальбомы |
| `GET /api/team/` | Команда проекта |
| `GET /api/achievements/` | Показатели для блока «Наше влияние» |
| `GET /api/organization/` | Информация об организации + история |

API отдаёт только записи с `is_visible = true`, изображения — абсолютным URL.

## Новости (ручное создание)

Новости создаются вручную в админке: раздел «Новости» → «Добавить».
Текст и фото переносятся из ВКонтакте руками (копируете текст, прикладываете
изображения). Поле «Внешняя ссылка» — ссылка на оригинальный пост; галочка
«Импортировано из VK» — необязательная пометка для бейджа на сайте.
Авто-импорт через VK API не используется.

## Скрытие записей

Чтобы убрать запись с сайта — снимите галочку «Отображать на сайте»
(`is_visible = False`). Запись исчезнет из API, но останется в админке.

## Что куда заносить (по письму заказчика)

- **Забеги** → раздел «Забеги»: название, дата, статус, место; ссылка на
  регистрацию в «Ссылка на регистрацию» (russiarunning), ссылка на добро.рф
  в «Доп. внешняя ссылка»; привязка приютов и спонсоров.
- **Приюты** (priut32, domspaseniya, dogpatrolrostov и т.д.) → «Благополучатели»,
  ТГ/сайт в «Официальный сайт».
- **Спонсоры** (Grub's up, Арома Саше) → «Партнёры», ссылка в «Сайт партнёра».
- **Команда** → «Команда»: имя, роль, фото, описание.
- **Достижения** (сумма помощи, города, цифры) → «Достижения (Наше влияние)».
- **Контакты, цель, история** → «Информация об организации».

## Как обновить фронт после правок дизайна

Фронт лежит исходниками отдельно (`../frontend` или ваш репозиторий). После
изменений пересоберите его и положите результат в `static_site/`:

```bat
cd frontend
npm install
npm run build
:: затем скопировать содержимое frontend/dist в backend/static_site
```

## Структура

```
backend/
  manage.py
  requirements.txt
  .env.example
  config/        # настройки, urls (отдача сайта + API), wsgi/asgi
  content/       # приложение: модели, админка, сериализаторы, API
    management/commands/seed_demo.py
  static_site/   # собранный React-сайт (отдаётся Django)
  media/         # загруженные через админку изображения
```

## Продакшен (VPS ~250 ₽/мес)

Задайте `DJANGO_SECRET_KEY`, `DJANGO_DEBUG=0`, `DJANGO_ALLOWED_HOSTS`.
В DEBUG Django сам отдаёт `static_site/` и `media/`. На проде статику и медиа
лучше раздавать через nginx, а Django запускать через gunicorn/uwsgi.
