# Run With Love (сайт, админка и API)

Один Django-проект, который отдаёт сразу **сайт, админку и API** по одному адресу.
React-фронт собран в статику и лежит в `static_site/`, его отдаёт сам Django.

Стек: Python 3, Django 5, Django REST Framework, SQLite, Django Admin (тема Jazzmin).
Фронт: React + Vite + Tailwind (TypeScript), исходники в отдельном репозитории.

Сайт не собирает персональные данные. Нет регистрации, форм, личных кабинетов и
оплаты. Запись на забеги и пожертвования идут на внешних сервисах по ссылкам.

## Возможности

- Публичное REST API только на чтение для фронтенда.
- Админка `/admin` для наполнения сайта без программиста.
- Сущности: забеги и дистанции, новости с фото, партнёры, благополучатели,
  фотоальбомы, отчёты, команда, показатели «Наше влияние», информация об организации.
- Скрытие записей с сайта без удаления (поле `is_visible`).
- Человеко-понятные адреса (slug) с транслитерацией кириллицы, плюс SEO-поля.
- Импорт новостей из ВКонтакте через VK API (по токену), а также ручное создание.
- Документация API (Swagger) на `/api/docs/`.

## Быстрый старт (локально)

```bat
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
copy .env.example .env
python manage.py migrate
python manage.py createsuperuser
python manage.py seed_demo      :: необязательно, демо-данные
python manage.py runserver
```

На Linux/Mac активация venv: `source .venv/bin/activate`.

После запуска всё на одном адресе:

- Сайт: http://127.0.0.1:8000/
- Админка: http://127.0.0.1:8000/admin/
- API: http://127.0.0.1:8000/api/
- Документация API (Swagger): http://127.0.0.1:8000/api/docs/

## Публичное API (read-only, без авторизации)

| Endpoint | Описание |
|---|---|
| `GET /api/races/` | Забеги (с дистанциями, партнёрами, благополучателями) |
| `GET /api/news/` | Новости (с галереей изображений) |
| `GET /api/partners/` | Партнёры и инфопартнёры |
| `GET /api/beneficiaries/` | Благополучатели (фонды, приюты) |
| `GET /api/galleries/` | Фотоальбомы |
| `GET /api/reports/` | Отчёты проекта |
| `GET /api/team/` | Команда проекта |
| `GET /api/achievements/` | Показатели для блока «Наше влияние» |
| `GET /api/organization/` | Информация об организации (в т.ч. «О нас») |
| `GET /api/docs/` | Swagger UI (документация API) |
| `GET /api/schema/` | OpenAPI-схема |

API отдаёт только записи с `is_visible = true`. Изображения возвращаются абсолютным URL.

## Импорт новостей из ВКонтакте

Реализован импорт через VK API. Включается после того, как в переменные окружения
добавят токен `VK_ACCESS_TOKEN` (и при необходимости `VK_GROUP_DOMAIN`).

- В админке (раздел «Новости») есть кнопки «Загрузить новости из ВКонтакте»
  (последние посты сообщества) и «Импортировать пост по ссылке».
- Команда для расписания: `python manage.py import_vk`.
- Хранятся последние 10 импортированных постов. Ручное создание новостей доступно всегда.

Без токена кнопки и команда покажут понятную ошибку и ничего не сломают.

## Скрытие записей

Чтобы убрать запись с сайта, снимите галочку «Отображать на сайте» (`is_visible = False`).
Запись пропадёт из публичного API, но останется в админке.

## Что куда заносить

- **Забеги** в раздел «Забеги»: название, дата, статус, место; ссылка на регистрацию,
  при необходимости другие ссылки; привязка партнёров и благополучателей; дистанции.
- **Благополучатели** (фонды, приюты) в раздел «Благополучатели», сайт или ТГ в «Официальный сайт».
- **Партнёры и спонсоры** в раздел «Партнёры», ссылка в «Сайт партнёра».
- **Новости** в раздел «Новости»: текст, дата, фото, при желании ссылка.
- **Фотоальбомы** в раздел «Фотоальбомы», с привязкой к забегу или без.
- **Отчёты** в раздел «Отчёты»: заголовок, описание, файл или ссылка.
- **Команда** в раздел «Команда»: имя, роль, фото, описание.
- **Показатели** (суммы, города, цифры) в «Достижения (Наше влияние)».
- **Контакты, цель, история** в «Информация об организации» (после заполнения на сайте
  появятся разделы «О нас» и «Контакты»).

Разделы «О нас» и «Команда» появляются на сайте только когда в них есть данные.

## Переменные окружения

Скопируйте `.env.example` в `.env` и при необходимости заполните:

| Переменная | Назначение |
|---|---|
| `DJANGO_SECRET_KEY` | Секретный ключ Django (на проде задайте свой). |
| `DJANGO_DEBUG` | `1` для разработки, `0` для продакшена. |
| `DJANGO_ALLOWED_HOSTS` | Разрешённые хосты (на проде ваш домен). |
| `CSRF_TRUSTED_ORIGINS` | Доверенные origin для входа в админку по https. |
| `SQLITE_PATH` | Путь к файлу базы (для хранения на постоянном диске). |
| `MEDIA_ROOT` | Путь к загруженным фото (для постоянного диска). |
| `VK_ACCESS_TOKEN` | Токен VK API для импорта новостей (необязательно). |
| `VK_GROUP_DOMAIN` | Короткое имя сообщества VK (по умолчанию `runwithlove`). |

## Деплой на Railway

Проект готов к запуску на Railway (есть `Procfile`, `requirements.txt` с gunicorn и whitenoise).

1. Создайте свой репозиторий с этим кодом на GitHub.
2. На railway.app: New Project, затем Deploy from GitHub repo, выберите репозиторий.
   Если код лежит во вложенной папке, укажите Root Directory.
3. Откройте сервис, вкладка Variables, добавьте переменные:
   ```
   DJANGO_SECRET_KEY = длинная случайная строка
   DJANGO_DEBUG = 0
   DJANGO_ALLOWED_HOSTS = *
   CSRF_TRUSTED_ORIGINS = https://*.up.railway.app
   DJANGO_SUPERUSER_USERNAME = admin
   DJANGO_SUPERUSER_EMAIL = admin@example.com
   DJANGO_SUPERUSER_PASSWORD = ваш пароль для входа в админку
   SQLITE_PATH = /app/data/db.sqlite3
   MEDIA_ROOT = /app/data/media
   ```
4. Добавьте Volume с точкой монтирования `/app/data`. Это нужно, чтобы база данных
   и загруженные фото не пропадали при обновлениях. Без Volume данные сбрасываются
   при каждом деплое.
5. Settings, затем Networking, затем Generate Domain. Получите адрес сайта.

После деплоя `Procfile` сам применит миграции, соберёт статику и создаст
администратора из переменных `DJANGO_SUPERUSER_*`. Логин и пароль администратора
задаёте вы в переменных окружения.

## Как обновить фронт после правок дизайна

Исходники фронта лежат в отдельном репозитории. После изменений соберите его и
положите результат в `static_site/`:

```bat
npm install
npm run build
:: затем скопировать содержимое dist в backend static_site
::   dist/index.html, dist/favicon.svg, dist/icons.svg  ->  static_site/
::   dist/assets/*                                       ->  static_site/assets/ (старое удалить)
```

После этого закоммитьте изменения и задеплойте бэкенд, сайт обновится.

```
  services/vk_api.py

```
